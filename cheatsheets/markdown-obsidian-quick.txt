║ MARKDOWN BEST PRACTICES                                      ║
║ • Use ATX-style headers (# ## ###) not underline style       ║
║ • Leave blank line before/after headers                      ║
║ • Use reference-style links for repeated URLs                ║
║ • Indent nested lists with 2 or 4 spaces (be consistent)     ║
║ • Use fenced code blocks (```) not indented                  ║
║ • Always specify language in code blocks                     ║
║ • Use | for tables, not HTML                                 ║
║ • Escape special characters when needed: \* \_ \` \#         ║
